# Portable user data (share & restore)

My Cosmos keeps a **working copy** in the browser (**IndexedDB**) for speed. Your **canonical** format is **JSON on disk**: one file per user under `data/`. To point the local server at another folder, set **`MY_COSMOS_DATA`** (legacy: `PROGRESS_TRACKER_DATA`).

## How to open the app

| Method | How | Saves go to |
|--------|-----|-------------|
| **Start.command** (recommended) | Double-click `start.command` in the app folder | `data/` folder on disk via local server |
| **index.html + folder picker** | Double-click `index.html`; click "Pick data/ folder" if prompted | `data/` folder on disk via File System Access API |
| **index.html only** | Double-click `index.html` | IndexedDB only (limited by browser quota) |

## Folder layout

| Path | Purpose |
|------|---------|
| `data/{username}_data.json` | **Main portable file** for that user |
| `data/file_boot.js` | Auto-generated preload for `file://` mode |
| `data/savepoints/` | Checkpoints created by the server (Ctrl+S via localhost) |
| `data/users_index.json` | Generated by `prepare.js` for `file://` user discovery |

## What counts as a full backup?

Each user file is named **`{username}_data.json`** and contains:

- `nodes` (array)
- `edges` (array)
- `displayName`, `workspaces`, `lastSaved`, and other app fields

**One file = one complete backup.** Savepoints are optional history.

## Three ways to keep data on disk

### 1. Start.command (best â€” zero friction)

Double-click `start.command`. Saves go to `data/` as real JSON files. You can zip `data/`, copy it, or sync it via Dropbox.

### 2. Chrome/Edge â€” pick a folder (no server needed)

When you open via `index.html` and browser storage is full, the app prompts you to pick a folder. Choose the `data/` folder. Saves write `{username}_data.json` and `file_boot.js` directly to disk.

### 3. Any browser â€” export/import

- **Export All**: downloads one JSON per user.
- **Import**: select `*_data.json` files to load into the browser.

## Recommended workflow

| Goal | Action |
|------|--------|
| Daily use | Open via `start.command` or `index.html` with folder picker |
| Backup | Copy `data/{username}_data.json` somewhere safe |
| Move to another computer | Copy `data/` folder â†’ open the app on the new machine |
| Share with someone | Send `{username}_data.json`; they use **Import** |

## Mental model

- **JSON on disk** = portable source of truth you can move anywhere.
- **Browser storage (IndexedDB)** = local cache for speed; not the only copy for large datasets.
